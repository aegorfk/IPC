# Калькулятор индексации, пеней и среднего заработка

Расчёта индексации зарплаты, индексации невыплаченных сумм, материальной ответственности работодателя и среднего заработка — прямо в Google Sheets.

---

## Что это и зачем

Когда работодатель недоплатил зарплату, возникает эффект домино. Деньги обесценились из-за инфляции, и работник вправе требовать большего. 
Этот скрипт автоматизирует взаимосвязанные расчёты, которые иначе приходится делать вручную:

**1. Индексация зарплаты** — по методологии [Основ законодательства Союза ССР и республик об индексации доходов населения от 25 июня 1991 г. N 2266-I](https://www.consultant.ru/cons/cgi/online.cgi?req=doc&base=ESU&n=5354#YuqMsZU8Pb8B4Wxn). 

**2. Индексация самой недоплаты** — ст.208 ГПК РФ по аналогии.
Право работника на индексацию денег независимо от начисленных процентов прямо подтверждено [п. 55 Постановления Пленума Верховного Суда РФ от 17.03.2004 № 2](https://www.consultant.ru/document/cons_doc_LAW_47257/9e80475e5cfe2f835c2624c2629eb460d2141eab/):
> *«Начисление процентов в связи с несвоевременной выплатой заработной платы не исключает права работника на индексацию сумм задержанной заработной платы в связи с их обесцениванием вследствие инфляционных процессов.»*

**3. Материальная ответственность работодателя (ст. 236 ТК РФ)** — денежная компенсацияи, начисляется на все: оклад, премии, отпускные, вынужденный прогул, выходное пособие, окончательный расчет.

**4. Средний заработок** для отпускных и выходного пособия — по **Постановлению Правительства РФ № 922** (до 01.09.2025) и **№ 540** (с 01.09.2025), с реконструкцией 12-месячной базы из корректных начислений. Восстанавливаются суммы отпускных, окнчательного расчета.
ИПЦ берётся из справочников Росстата через калькулятор КонсультантПлюс.

---

## Что умеет скрипт

### Индексация недоплаты по ИПЦ
- Для каждой строки недоплаты (оклад, ежемесячные / квартальные / годовые премии, отпускные) считает цепной множитель ИПЦ от месяца недоплаты до даты подачи иска (указывается в колонке сверху)
- Дефляционные месяцы (ИПЦ < 100%) приводятся к 1,0
- Дата начала периода: первая дата выплаты зарплаты (восстаналвивается из фактических расчетных листов) с переносом на ближайший рабочий день по производственному календарю
- Суммы уменьшаются пропорционально отработанным дням
- Если данных Росстата за последние месяцы ещё нет — скрипт считает по опубликованным и ставит примечание в ячейку

### Материальная ответственность (ст. 236 ТК РФ)
- Начисляется на оклад, все виды премий, отпускные, выходное пособие, вынужденный прогул
- Период: со следующего дня после установленной даты выплаты до даты окончания расчёта
- При изменении ключевой ставки период разбивается на интервалы

### Реконструкция среднего заработка
- Для листа `Отпуска` автоматически собирает корректную 12-месячную базу из листов `Оклад`, `Ежемесячные`, `Ежеквартальные`, `Ежегодные`
- Годовые премии включаются только за завершённый календарный год
- Переходная дата ПП № 922 → ПП № 540: 01.09.2025 (в расчет стали входить не только стимулирующие премии, но и поощрительные)

### Импорт расчётных листков (работает стабильно с 1С:ЗУП)
- Сканирует папку Google Drive, указанную пользователем по ссылке, с форматами: Google Sheets, Google Docs, HTML, CSV, PDF, JPG
- OCR и VLM распознавание через Polza.ai (по умолчанию `google/gemini-3.1-flash-lite`)
- Если суммы после парсинга не сходятся с итогами расчётного листка — автоматически перечитывает через более сильную модель
- Нормализует строки по категориям: оклад, ежемесячные / квартальные / годовые премии, отпуска, выходное пособие, удержания и др.

## Беклог (в разработке)

- [ ] Сверка сумм с данными 2-НДФЛ
- [ ] Сверка начислений и выплат с датами выплаты зарплаты — для выявления задержек без компенсации
- [ ] Вкладка с годовой методологией расчёта индексации (с переключателем учёта премий)
- [ ] Начисление матответственности на саму сумму недоплаты (отдельной строкой)
- [ ] Разбивка выплат на две части месяца (аванс / окончательный расчёт)
- [ ] Отпуска за время вынужденного прогула

## Структура листов

### Рабочие листы

| Лист | Что считает |
|------|-------------|
| `Оклад` | Недоплата по окладу, ИПЦ помесячно, индексация, пени |
| `Ежемесячные` | Недоплата по ежемесячным премиям, индексация, пени |
| `Ежеквартальные` | Недоплата по квартальным премиям, индексация, пени |
| `Ежегодные` | Недоплата по годовым премиям, индексация, пени |
| `Отпуска` | Недоплата по отпускным, реконструкция базы среднего заработка, индексация, пени |

### Служебные листы (создаются автоматически) - будут убраны в слудеющих версиях

| Лист | Содержимое |
|------|-----------|
| `Расчетные_листы` | Нормализованные строки из расчётных листков |
| `Расчетные_листы_Свод` | Агрегированная сводка по файлу / месяцу / категории |
| `Расчетные_листы_Диагностика` | Сверка импорта с рабочими листами |
| `Расчетные_листы_Качество` | Отчёт по каждому расчётному листку, предупреждения |
| `Расчетные_листы_Состояние` | Индекс последнего импорта (не перечитывает неизменившиеся файлы) |
| `Расчетные_листы_VLM` | Журнал VLM-fallback с токенами, стоимостью и сырым JSON |
| `Реконструкция_Оклад` / `Реконструкция_*` | Структурные вкладки для реконструкции из расчётных листков |

## Установка

1. Откройте таблицу Google Sheets
2. `Расширения` → `Apps Script`
3. Скопируйте файлы скрипта из репозитория
4. В `appsscript.json` убедитесь, что включены scopes:
   ```
   https://www.googleapis.com/auth/spreadsheets
   https://www.googleapis.com/auth/documents
   https://www.googleapis.com/auth/drive
   https://www.googleapis.com/auth/drive.readonly
   https://www.googleapis.com/auth/script.external_request
   ```
5. Подключите Advanced Drive service (`Drive API v2`)
6. Для VLM-fallback добавьте в `Project Settings → Script properties` (ВАШ API ключ от агрегатора нейронок для распознавания):
   ```
   POLZA_API_KEY = <ваш ключ>
   ```

## Меню в таблице

После установки появляются два меню:

### `Индексация недоплаты`
- **Обновить активный лист** — пересчитать текущий лист
- **Обновить все листы** — последовательно: Оклад → Ежемесячные → Ежеквартальные → Ежегодные → Отпуска
- **Обновить ежемесячные / квартальные / ежегодные премии** — отдельный пересчёт листа
- **Обновить отпуска** — пересчитать лист Отпуска

### `Импорт расчетных листов`
- **Проверить импорт без перезаписи** — обновить только отчёт качества
- **Импортировать расчётные листки** — читать папку Drive, обновить `Расчетные_листы` и `Расчетные_листы_Свод`
- **Полный импорт с перечитыванием** — принудительно перечитать все файлы
- **Проверить VLM настройки** — показать статус API-ключа и используемую модель
- **Создать все вкладки структуры расчетных листов** — создать `Реконструкция_*`
- **Заполнить вкладки реконструкции из импорта** — перенести данные из `Расчетные_листы` в структурные вкладки
- **Очистить импорт расчетных листов** — удалить все служебные листы импорта

## Источники данных

- **ИПЦ**: справочники КонсультантПлюс (`indexes.json`, `indexes_from_2022.json`) через `UrlFetchApp`
- **Производственный календарь**: `calendar.json` КонсультантПлюс (нерабочие дни для переноса даты выплаты)
- **Ключевая ставка**: `rates.json` КонсультантПлюс (история ставок для расчёта пеней)
- **Регион по умолчанию**: Российская Федерация (`id: 1`)
