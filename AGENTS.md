# Project Instructions

## Актуальная документация

Всегда используй Context7, когда нужен актуальный справочник по библиотеке или API, примеры кода, шаги установки или конфигурации. Для этого сначала вызывай `resolve-library-id`, затем `query-docs`, даже если пользователь явно не написал `use context7`.

## Универсальный импорт расчетных листов

Цель проекта — создать максимально универсальный механизм считывания и нормализации расчетных листов.

- 1С:ЗУП — наиболее часто встречающийся и приоритетный для поддержки формат расчетных листов, который следует использовать как основной практический сценарий и источник тестовых примеров.
- При этом не завязывать импорт, распознавание и внутреннюю модель данных исключительно на 1С:ЗУП, ее названия разделов, структуру выгрузки или конкретный вариант расчетного листка.
- Поддерживать расчетные листы из разных учетных систем и источников при наличии достаточных данных для определения периода, начислений, выплат, удержаний и иных расчетных показателей.
- Термины и структуры конкретного источника преобразовывать в единую внутреннюю модель на уровне адаптеров и правил распознавания.
- Новый функционал импорта проектировать как независимый от поставщика исходных документов, сохраняя полноценную и приоритетную поддержку расчетных листов 1С:ЗУП.
- Расчеты требований, индексации и материальной ответственности строить по реконструированным денежным потокам, восстановленным из расчетных листков и методологии расчета, а не по фактическим банковским выплатам. Фактические выплаты использовать только как источник данных для реконструкции, если это явно предусмотрено логикой.

## Таблица и Docs-расшифровка

- Google Sheets остается основным рабочим местом расчета: функции пересчета должны записывать итоговые суммы и рабочие значения обратно в существующую структуру таблицы, не создавая параллельную замену без явного запроса.
- Google Docs используется для раскрытия и текстовой расшифровки расчета по уже рассчитанным данным таблицы: формулы, дневные/помесячные строки, методология, пояснения и итоговая судебная narrative-структура.
- Нельзя подменять расчет в таблице генерацией Docs: если пользователь просит пересчитать вынужденный прогул, материальную ответственность или накопленные отпуска, сначала обновляются соответствующие ячейки таблицы, а Docs только детализирует результат.
- Нельзя очищать или перестраивать весь Google Doc без явного запроса. Автоматически обновляемые фрагменты должны быть ограничены маркерами или иным локальным диапазоном, чтобы сохранять существующую структуру документа.
